## Standing rule: keep the SSOT docs current
Whenever a decision, plan, feature spec, scope change, or idea evolves — in conversation OR in files — update the relevant SSOT doc (`PROJECT.md` for the platform, `GLASSES.md` for the glasses; if it touches both, update both) in the same turn:
- Edit the relevant section when a spec changes.
- Append a dated entry to the **Decision Log** when an idea evolves (PROJECT.md uses `Dx`, GLASSES.md uses `Gx`); reference the section it changed and cross-link the other doc when relevant.
- Don't delete superseded reasoning — mark it "superseded by Dx/Gx" so the trail survives.
- Bump the "Last updated" date at the top.

This is a semantic update only the model can do (not a shell hook). Treat it as a default working behavior, not something to be asked for each time.
