**[PROJECT.md](PROJECT.md) is the single source of truth.** Read it at the start of any session before doing project work.

## Standing rule: keep PROJECT.md current
Whenever a decision, plan, feature spec, scope change, or idea evolves — in conversation OR in files — update `PROJECT.md` in the same turn:
- Edit the relevant section when a spec changes.
- Append a dated entry to the **Decision Log** (§10) when an idea evolves; reference the section it changed.
- Don't delete superseded reasoning — mark it "superseded by Dx" so the trail survives.
- Bump the "Last updated" date at the top.

This is a semantic update only the model can do (not a shell hook). Treat it as a default working behavior, not something to be asked for each time.
